# دليل نظام إدارة علاقات العملاء (CRM)

## D-Arrow Business — شرح كامل بالعربية

---

## 1. ما هو هذا النظام؟

نظام **CRM** (Customer Relationship Management) هو وحدة مدمجة داخل تطبيق D-Arrow Business لإدارة دورة المبيعات من أول تواصل مع عميل محتمل حتى إغلاق الصفقة. يعمل النظام على نمط **Zoho CRM** و **Odoo** من حيث:

- **عرض المسار (Pipeline)** كلوحة كانبان مع سحب وإفلات
- **صفحة السجل** بتخطيط من عمودين: بيانات السجل + لوحة **المحادثة (Chatter)**
- **شريط المراحل** لتغيير حالة العميل أو الصفقة بنقرة واحدة
- **بحث موحّد** عبر العملاء المحتملين وجهات الاتصال والصفقات
- **إشعارات** عند التعيين أو تغيير مرحلة الصفقة

البيانات تُخزَّن في **Firebase Firestore** حسب الشركة (Company-scoped)، وكل مستخدم يرى ما يسمح له دوره برؤيته.

---

## 2. كيفية الدخول إلى CRM

1. سجّل الدخول إلى التطبيق.
2. من القائمة الجانبية اختر **CRM**.
3. ستظهر شريط فرعي أعلى الصفحة يحتوي على:
   - لوحة التحكم
   - العملاء المحتملون
   - جهات الاتصال
   - الصفقات
   - مهام CRM
   - التقارير
4. على يمين الشريط: **بحث CRM** و**جرس الإشعارات**.

### المسارات (Routes)

| المسار | الوظيفة |
|--------|---------|
| `/crm` | لوحة التحكم والإحصائيات |
| `/crm/leads` | العملاء المحتملون (مسار / قائمة) |
| `/crm/leads/:id` | تفاصيل عميل محتمل |
| `/crm/contacts` | جهات الاتصال |
| `/crm/contacts/:id` | تفاصيل جهة اتصال |
| `/crm/deals` | الصفقات (كانبان / قائمة) |
| `/crm/deals/:id` | تفاصيل صفقة |
| `/crm/tasks` | مهام المتابعة |
| `/crm/reports` | التقارير |

---

## 3. لوحة التحكم (`/crm`)

تعرض نظرة عامة على أداء المبيعات:

### بطاقات المؤشرات (KPIs)

- إجمالي العملاء المحتملين
- إجمالي جهات الاتصال
- الصفقات النشطة (غير فائزة/خاسرة)
- الصفقات الفائزة والخاسرة
- **الإيراد المتوقع** (مجموع قيمة الصفقات × نسبة الاحتمال)
- **معدل التحويل** (نسبة جهات الاتصال من إجمالي العملاء المحتملين)

### الرسوم البيانية

- **قمع المبيعات**: عملاء → مؤهلون → عرض → تفاوض → فوز
- **العملاء حسب المصدر**: موقع، إحالة، تواصل اجتماعي، إلخ
- **الصفقات حسب المرحلة**
- **الإيرادات حسب الشهر** (من الصفقات الفائزة)

---

## 4. العملاء المحتملون (Leads)

العميل المحتمل هو شخص أو شركة **لم تُحوَّل بعد** إلى جهة اتصال رسمية.

### طريقتان للعرض

| الوضع | الوصف |
|-------|--------|
| **مسار (Pipeline)** — الافتراضي | لوحة كانبان؛ كل عمود = مرحلة. اسحب البطاقة لتغيير المرحلة تلقائياً |
| **قائمة (List)** | جدول مع بحث، تصفية، ترتيب، وترقيم صفحات |

**التبديل:** أزرار **قائمة | مسار** أعلى الصفحة.

### مراحل العميل المحتمل

```
جديد → تم التواصل → مؤهل → عرض مُرسل → تفاوض → فوز / خسارة
```

| المرحلة | المعنى |
|---------|--------|
| جديد | تم إدخاله ولم يُتواصل معه |
| تم التواصل | تم أول تواصل |
| مؤهل | يستوفي معايير العميل المناسب |
| عرض مُرسل | أُرسل عرض سعر أو propuesta |
| تفاوض | مرحلة التفاوض على الشروط |
| فوز | تحوّل أو أُغلق بنجاح |
| خسارة | لم يستمر |

### إضافة عميل محتمل

الحقول: الاسم، الهاتف، البريد، الشركة، المصدر، الأولوية، المستخدم المسنود، ملاحظات.

- يتم **التحقق من البيانات** عبر Zod قبل الحفظ.
- عند الإنشاء يُسجَّل نشاط تلقائي: «تم إنشاء العميل المحتمل».
- إذا عُيِّن لمستخدم آخر، يصله **إشعار**.

### صفحة تفاصيل العميل المحتمل

#### أعلى الصفحة

- **مسار التنقل**: CRM / العملاء المحتملون / الاسم
- **شريط المراحل**: انقر على مرحلة لتغييرها
- **أزرار ذكية**: عدد المهام، الملاحظات، المرفقات
- **تحويل إلى جهة اتصال**: يحوّل العميل وينقل الملاحظات والأنشطة والمهام والمرفقات

#### العمود الأيسر (بيانات السجل)

- معلومات الاتصال والشركة والمسؤول
- قائمة مهام CRM المرتبطة

#### العمود الأيمن (المحادثة — Chatter)

مثل Odoo:

- **تسجيل ملاحظة** داخلية
- **جدولة نشاط**: مكالمة، بريد، اجتماع
- **رفع مرفق** (صور، PDF، Excel، Word) عبر Firebase Storage
- **سجل موحّد** يجمع الملاحظات والأنشطة زمنياً

### التصفية في وضع القائمة

- البحث النصي
- الحالة (متعدد)
- المصدر
- المسؤول (مسند / غير مسند / مستخدم محدد)
- نطاق التاريخ

---

## 5. تحويل العميل المحتمل إلى جهة اتصال

عملية أساسية في CRM (مثل Zoho/Odoo):

1. افتح صفحة العميل المحتمل.
2. اضغط **«تحويل إلى جهة اتصال»**.
3. يقوم النظام تلقائياً بـ:
   - إنشاء **جهة اتصال** (الاسم، البريد، الهاتف، الشركة)
   - **نقل** الملاحظات، الأنشطة، المهام، والمرفقات إلى جهة الاتصال
   - تحديث حالة العميل إلى **فوز**
   - تسجيل نشاط: «تم تحويل العميل إلى جهة اتصال»
4. يُوجَّهك إلى صفحة جهة الاتصال الجديدة.

> **ملاحظة:** لا يمكن تحويل نفس العميل مرتين؛ بعد التحويل يظهر زر «عرض جهة الاتصال».

---

## 6. جهات الاتصال (Contacts)

جهة الاتصال هي **عميل مؤكّد** في قاعدة بياناتك (بعد التحويل أو الإضافة اليدوية).

### القائمة

- بحث، ترتيب، ترقيم صفحات
- فلتر حسب المسؤول

### صفحة التفاصيل

| القسم | المحتوى |
|-------|---------|
| نظرة عامة | الاسم، البريد، الهاتف، الوظيفة، القسم، الحساب |
| الصفقات | الصفقات المرتبطة بجهة الاتصال |
| المهام | مهام CRM المرتبطة |
| الملاحظات | إنشاء / تعديل / حذف |
| المرفقات | رفع ومعاينة وتحميل |
| الجدول الزمني | كل الأنشطة مسجّلة |

---

## 7. الصفقات (Deals)

الصفقة تمثل **فرصة بيع** بقيمة مالية ومراحل محددة.

### طريقتان للعرض

| الوضع | الوصف |
|-------|--------|
| **مسار (Kanban)** — الافتراضي | أعمدة حسب المرحلة؛ اسحب الصفقة لتغيير مرحلتها |
| **قائمة** | جدول: العنوان، جهة الاتصال، المرحلة، القيمة، الاحتمال |

### مراحل الصفقة

```
عميل محتمل → تم التواصل → مؤهل → عرض مُرسل → تفاوض → فوز / خسارة
```

عند تغيير المرحلة يُحدَّث **احتمال الإغلاق** تلقائياً:

| المرحلة | الاحتمال الافتراضي |
|---------|---------------------|
| عميل محتمل | 10% |
| تم التواصل | 20% |
| مؤهل | 40% |
| عرض مُرسل | 60% |
| تفاوض | 80% |
| فوز | 100% |
| خسارة | 0% |

### حقول الصفقة

- العنوان
- جهة الاتصال
- القيمة والعملة
- المرحلة والاحتمال
- تاريخ الإغلاق المتوقع
- المستخدم المسنود

### صفحة تفاصيل الصفقة

- شريط مراحل قابل للنقر
- أزرار ذكية (مهام، ملاحظات، مرفقات)
- تخطيط **سجل + محادثة** مثل صفحة العميل المحتمل
- عند **الفوز** أو **الخسارة** يُرسل إشعار للمسؤول

---

## 8. مهام CRM

مهام المتابعة مرتبطة بعميل محتمل أو جهة اتصال أو صفقة.

### أنواع المهام

| النوع | الاستخدام |
|-------|-----------|
| مكالمة | متابعة هاتفية |
| اجتماع | موعد مع العميل |
| بريد | مراسلة بريدية |
| متابعة | Follow-up عام |
| عرض | إرسال أو متابعة عرض |

### حالات المهمة

```
قيد الانتظار → قيد التنفيذ → مكتملة / ملغاة
```

### صفحة المهام (`/crm/tasks`)

- **اليوم**: مهام مستحقة اليوم
- **قادمة**: مهام بتاريخ لاحق
- **متأخرة**: مهام تجاوزت تاريخ الاستحقاق
- جدول كامل مع فلتر حسب الحالة

---

## 9. الملاحظات والمرفقات والأنشطة

### الملاحظات

- تُربَط بعميل / جهة اتصال / صفقة
- يمكن **إنشاؤها، تعديلها، وحذفها**
- إضافة ملاحظة تُسجَّل في الجدول الزمني

### المرفقات

- رفع عبر **Firebase Storage**
- الأنواع المدعومة: JPG, PNG, GIF, WebP, PDF, Excel, Word
- معاينة للصور، تحميل للملفات، حذف

### الأنشطة (الجدول الزمني)

يُسجَّل النظام تلقائياً:

| الحدث | متى |
|-------|-----|
| إنشاء عميل محتمل | عند الإضافة |
| تحديث عميل | عند تعديل الحقول |
| تحويل إلى جهة اتصال | عند التحويل |
| إنشاء / تحديث جهة اتصال | CRUD |
| إنشاء / تغيير مرحلة صفقة | CRUD + سحب كانبان |
| إنشاء / إكمال مهمة | CRUD |
| إضافة ملاحظة | عند النشر |
| رفع ملف | عند الرفع |

يمكن أيضاً **تسجيل نشاط يدوي** (مكالمة، اجتماع، بريد) من لوحة المحادثة.

## 10. الإشعارات

الجرس في شريط CRM يعرض:

- تعيين عميل محتمل
- تعيين صفقة
- تعيين مهمة
- صفقة فائزة أو خاسرة

**الإجراءات:**

- النقر على إشعار → الانتقال للسجل + تعليم كمقروء
- «تعليم الكل كمقروء»

## 11. التقارير (`/crm/reports`)

### تقارير العملاء المحتملين

- إجمالي العملاء
- معدل التحويل
- توزيع المصادر

### تقارير المبيعات

- صفقات فائزة / خاسرة
- إجمالي الإيراد من الصفقات الفائزة

### أداء الفريق

جدول يُظهر لكل موظف:

- عدد الصفقات الفائزة
- الإيراد
- المهام المكتملة
- معدل تحويل العملاء المحتملين

---

## 12. الصلاحيات والأدوار

| الدور | ما يمكنه |
|-------|----------|
| **Super Admin / Admin** | كل شيء: عرض، إنشاء، تعديل، حذف |
| **Manager** | إدارة كاملة للـ CRM (بدون حذف عام في بعض الحالات) |
| **Employee** (مبيعات) | يرى فقط السجلات **المسندة إليه** |
| **Viewer** | عرض فقط |

### قواعد الوصول

- موظف المبيعات يرى العملاء والصفقات والمهام المربوطة بـ `assignedTo` أو `assigneeId` الخاص به.
- المدير والمسؤول يريان **كل** سجلات الشركة.

## 13. البحث الموحّد

حقل **«بحث CRM…»** في الشريط العلوي:

- ابحث بـ **حرفين على الأقل**
- يبحث في: اسم العميل، الشركة، البريد، اسم جهة الاتصال، عنوان الصفقة
- النتائج تظهر في قائمة منسدلة → انقر للانتقال مباشرة

## 14. كيف تُخزَّن البيانات تقنياً؟

كل بيانات CRM تابعة للشركة في Firestore:

```
companies/{companyId}/
├── leads              ← العملاء المحتملون
├── contacts           ← جهات الاتصال
├── deals              ← الصفقات
├── crm_tasks          ← مهام CRM
├── notes              ← الملاحظات
├── activities         ← الأنشطة / الجدول الزمني
├── attachments        ← المرفقات (روابط Storage)
└── crm_notifications  ← الإشعارات
```

الملفات الفعلية في:

```
Firebase Storage → crm/{companyId}/{entityType}/{entityId}/...
```

## 15. دورة حياة المبيعات الكاملة (مثال عملي)

```
1. إضافة عميل محتمل (مصدر: الموقع)
        ↓
2. سحبه في المسار: جديد → تم التواصل → مؤهل
        ↓
3. تسجيل مكالمة في المحادثة + رفع عرض PDF
        ↓
4. إنشاء مهمة «متابعة بعد 3 أيام»
        ↓
5. تحويل إلى جهة اتصال
        ↓
6. إنشاء صفقة بقيمة 50,000 USD
        ↓
7. سحب الصفقة في كانبان: مؤهل → عرض مُرسل → تفاوض → فوز
        ↓
8. مراجعة الإيراد في لوحة التحكم والتقارير
```

---

## 16. نصائح للاستخدام اليومي

1. **ابدأ من المسار (Pipeline)** للعملاء والصفقات — أسرع من الجداول.
2. **استخدم المحادثة (Chatter)** لكل تواصل — يبني سجل كامل للفريق.
3. **عيِّن المسؤول** لكل سجل — يفعّل الإشعارات ويحترم صلاحيات موظف المبيعات.
4. **حوِّل العميل** فقط عند التأكد — بعدها يصبح جهة اتصال دائمة.
5. **راجع لوحة التحكم** صباحاً و**المهام المتأخرة** في `/crm/tasks`.

---

## 17. الأسئلة الشائعة

**س: لماذا لا أرى بعض العملاء أو الصفقات؟**  
ج: إذا كان دورك «موظف»، ترى فقط ما هو مسند إليك. اطلب من المدير إعادة التعيين.

**س: ما الفرق بين العميل المحتمل وجهة الاتصال؟**  
ج: العميل المحتمل = فرصة قيد التقييم. جهة الاتصال = عميل مؤكّد في قاعدة البيانات.

**س: هل التغيير في المسار يحفظ تلقائياً؟**  
ج: نعم، عند سحب البطاقة بين الأعمدة يُحدَّث Firestore فوراً ويُسجَّل نشاط.

**س: أين المرفقات؟**  
ج: في لوحة المحادثة على يمين صفحة السجل، أو في تبويب المرفقات لجهات الاتصال.

**س: كيف أتابع أداء الفريق؟**  
ج: من **CRM → التقارير → أداء الفريق**.

|--------|-------|---------------|

| العملاء المحتملون | فرص أولية | كانبان / قائمة |
| جهات الاتصال | عملاء مؤكّدون | قائمة + تفاصيل |
| الصفقات | فرص بيع | كانبان / قائمة |
| مهام CRM | متابعات | لوحة + جدول |
| التقارير | تحليل | جداول وإحصائيات |
